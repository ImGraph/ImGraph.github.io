Creer une livraison

Dans cette section, on va voir comment créer, distribuer et identifier une livraison.


Le but de passer par le site est de standardiser et suivre les différentes livraisons de Surans.
L'avantage est donc de ne pas avoir une livraison chez le client dont on n'arrive pas à retrouver les conditions de compilation.
Le serveur TeamCity enregistre toute l'étape de compilation (et donc les différentes versions des librairies utilisées) ce qui permettra, même 6 mois après, de pouvoir retrouver les conditions de développement de l'époque où la version a été envoyée.

Créer une livraison

Nous avons vu à la page TeamCity que le serveur TeamCity recompile et créé une nouvelle livraison à chaque fois qu'un changement à lieu sur l'entrepot GIT. C'est cette livraison que nous allons mettre à disposition des partenaires.
Cependant, l'archive ainsi créée n'est pas directement accessible (pour des raisons de confidentialitées). Nous avons donc mis en place un site où nous pouvons rendre publique une livraison et où les douaniers peuvent télécharger les différentes livraisons ainsi créées.
La page de création de livraison est accessible ici et est assez simple à utiliser. Le script php se chargera d'envoyer un mail aux différents partenaires (liste modifiable à volonté), créera une nouvelle ligne en base de donnée pour identifier cette livraison et épinglera le numéro de build correspondant. Epingler un build revient à prévenir TeamCity qu'une livraison est spéciale et ne doit pas être supprimmée. En effet, après 30 jours, TeamCity supprime les anciennes livraisons qui n'ont pas été épinglées afin de réduire l'espace occupé par les différentes compilations.
La page de téléchargement est accessible ici et permet de télécharger une livraison particulière mais aussi d'envoyer l'identifiant d'une nouvelle machine.

Distribuer une livraison

Une fois l'archive téléchargée, les douaniers doivent décompresser l'archive sur le bureau, puis, après chaque génération de clé executer le script patch_key.bat. Ce script vérifiera que la version qui se trouve sur la clé n'est pas plus récente et si besoin remplacera la version sur la clé.

Identifier une version livrée

Après les retours de mission, il est souvent necessaire de pouvoir identifier les versions utilisées par les douaniers pour pouvoir comprendre exactement ce qui s'est passé. Il suffit de regarde le début des logs, les versions du programme et du fichier suransParams.xml sont indiquées. Le numéro peut se trouver sous deux formes : 1.0.XXX et 1.1.XXX.
Si le numéro est de type 1.0.XXX, c'est que la livraison a été faite manuellement et ce numéro correspond au numéro indiqué dans CMakelist.txt.
Si le numéro est de type 1.1.XXX, c'est que la livraison a été faite correctement via le site et le dernier numéro (XXX) correspond au numéro de build dans teamCity.
